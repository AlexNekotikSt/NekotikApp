# Застосування патерну «Відвідувач» для обробки віджетів

## Мета

Розробити розширювану систему виконання операцій над віджетами без зміни їхньої внутрішньої структури.

Патерн **Visitor** дозволяє відокремити логіку обробки від самих класів віджетів, що особливо корисно для реалізації:

- звітності,
- експорту,
- валідації,
- збору статистики.


## Контекст використання

У системі наявні такі типи віджетів:

- `TextWidget`
- `NumericWidget`
- `DateWidget`
- `FileWidget`
- `PictureWidget`

Над цими віджетами періодично необхідно виконувати незалежні дії:

- Генерація звітів (наприклад, у PDF/Excel)
- Експорт у зовнішній формат (наприклад, JSON/XML)
- Підрахунок кількості медіа-файлів у `FileWidget` та `PictureWidget`
- Збір мета-даних (розміри, типи, статистика)

Ці дії не повинні змінювати самі класи віджетів.
